# Codex Hook Catalog

This folder contains the Codex-ready hook catalog generated from the `davila7/claude-code-templates` hook set.

Layout:
- `aitmpl-codex/catalog.json` indexes every bundle.
- `aitmpl-codex/<category>/<bundle>/` is one self-contained hook package.
- Each bundle includes:
  - `hooks.json`
  - `.codex/hooks/` support files when needed
  - `bundle.json`
  - `README.md`

Install one bundle at a time:
```bash
python3 hooks/aitmpl-codex/install-bundle.py hooks/aitmpl-codex/<category>/<bundle> <target-repo>
```

If you prefer manual install:
1. Copy the bundle directory into your repo.
2. Merge the bundle `hooks.json` into `<repo>/.codex/hooks.json`.
3. Copy the bundle `.codex/hooks/` directory into `<repo>/.codex/hooks/`.
4. Restart Codex.

For testing:
- `aitmpl-codex/tools/harness.py` installs each bundle into a temp workspace and runs a dry harness against it.
- `aitmpl-codex/tools/report.json` is generated by the harness and should not be committed.

Compatibility:
- Some bundles are `direct` Codex matches.
- Others are `adapted` where Claude Code had an event Codex does not currently expose.
- Network/service hooks like Slack, Discord, Telegram, Vercel, and LangSmith are skipped by the harness and may need manual credentials.
